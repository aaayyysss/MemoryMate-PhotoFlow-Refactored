Google does not publish the exact production internals of Google Photos search. What can be reconstructed from Google’s public materials is a **hybrid retrieval architecture** with multiple candidate sources, followed by ranking, then confidence or presentation logic. Google has publicly described visual search as comparing images and ranking matches by similarity and relevance, and has also described multimodal search, text in images, and ranking as core ingredients of modern search. ([blog.google][1])

For MemoryMate, the strongest next move is not another model, it is this:

**candidate fusion plus reranking**

That means:

1. build candidates from several channels in parallel
2. merge them with evidence
3. rerank with family-aware weights
4. expose diagnostics and confidence

That is the pattern that will make your search much more robust.

## The simplified Google-Photos-style architecture for MemoryMate

**Stage A, Query Planning**

* Parse the query or preset into intent
* Resolve family: `type`, `people_event`, `scenic`, `animal_object`, `utility`
* Extract constraints: person, OCR text, screenshot intent, time, favorites, GPS

**Stage B, Candidate Generation**
Run multiple candidate builders in parallel, depending on family:

* `DocumentCandidateBuilder`
* `ScreenshotCandidateBuilder`
* `PeopleCandidateBuilder`
* `ScenicSemanticBuilder`
* `UtilityMetadataBuilder`

Each builder returns:

* candidate paths
* evidence per path
* rejection histogram
* builder confidence

**Stage C, Candidate Fusion**
Merge all candidates into one pool by path.
For each path, accumulate evidence:

* clip scores
* OCR hits
* screenshot evidence
* document evidence
* face evidence
* event evidence
* metadata boosts

**Stage D, Reranking**
Use a canonical ranker to compute final score from the fused evidence.
This is where you prevent split-brain logic.

**Stage E, Confidence and Presentation**
Evaluate:

* whether the right index was ready
* whether enough hard evidence survived
* whether the result set is trustworthy
* whether to show warning, empty-state honesty, or normal results

That overall structure is much closer to public Google-style retrieval systems than a single linear pipeline. Google has described ranking as a distinct layer after candidate understanding, and Lens-style search as image comparison plus relevance ranking, which maps well onto this candidate-fusion pattern. ([blog.google][2])

```python
import os
from typing import Dict

from services.candidate_builders.base_candidate_builder import BaseCandidateBuilder, CandidateSet
from services.query_intent_planner import QueryIntent
from logging_config import get_logger

logger = get_logger(__name__)

_SCREENSHOT_MARKERS = (
    "screenshot", "screen shot", "schirmfoto", "bildschirmfoto", "snap",
)

_UI_TERMS = (
    "whatsapp", "instagram", "battery", "lte", "5g", "wifi",
    "notification", "settings", "search", "cancel", "ok",
)


class ScreenshotCandidateBuilder(BaseCandidateBuilder):
    def build(
        self,
        intent: QueryIntent,
        project_meta: Dict[str, dict],
        limit: int = 500,
    ) -> CandidateSet:
        text_terms = intent.text_terms or []
        candidates = []
        evidence_by_path = {}
        rejection_counts = {}

        for path, meta in project_meta.items():
            is_screenshot = bool(meta.get("is_screenshot"))
            basename_lower = os.path.basename(path).lower() if path else ""
            filename_marker = any(m in basename_lower for m in _SCREENSHOT_MARKERS)

            ocr_text = (meta.get("ocr_text") or "").lower()
            ui_hit = any(t in ocr_text for t in _UI_TERMS) if ocr_text else False

            w = meta.get("width") or 0
            h = meta.get("height") or 0
            aspect = (max(w, h) / max(1, min(w, h))) if w and h else 0.0

            looks_like_phone_screen = (
                700 <= min(w, h) <= 1500
                and 1.7 <= aspect <= 2.4
                and (meta.get("face_count") or 0) == 0
            )

            term_hit = False
            if text_terms and ocr_text:
                term_hit = any(t.lower() in ocr_text for t in text_terms)

            accepted = is_screenshot or filename_marker or ui_hit or looks_like_phone_screen

            if accepted:
                evidence = {
                    "builder": "screenshot",
                    "is_screenshot_flag": is_screenshot,
                    "filename_marker": filename_marker,
                    "ui_text_hit": ui_hit,
                    "looks_like_phone_screen": looks_like_phone_screen,
                    "text_term_hit": term_hit,
                    "ocr_text_len": len(ocr_text),
                    "screenshot_score": (
                        0.50 * float(is_screenshot)
                        + 0.20 * float(filename_marker)
                        + 0.20 * float(ui_hit)
                        + 0.10 * float(looks_like_phone_screen)
                    ),
                }
                candidates.append(path)
                evidence_by_path[path] = evidence
            else:
                if (meta.get("face_count") or 0) > 0:
                    reason = "has_faces"
                elif min(w, h) > 0 and min(w, h) < 700:
                    reason = "too_small"
                else:
                    reason = "insufficient_evidence"
                rejection_counts[reason] = rejection_counts.get(reason, 0) + 1

            if len(candidates) >= limit:
                break

        logger.info(
            f"[ScreenshotCandidateBuilder] screenshots: "
            f"{len(candidates)}/{len(project_meta)} "
            f"(rejections={rejection_counts})"
        )

        return CandidateSet(
            family="type",
            candidate_paths=candidates,
            evidence_by_path=evidence_by_path,
            source_counts={"screenshot_candidates": len(candidates)},
            builder_confidence=0.85 if candidates else 0.0,
            ready_state="ready" if candidates else "empty",
            notes=[f"Screenshot builder: {len(candidates)} candidates"],
            diagnostics={"rejections": rejection_counts},
        )
```

### Why

Your current screenshot detection is mixed into document logic. That hurts precision. This split is one of the biggest quality upgrades.

```python
def _fuse_candidate_sets(self, candidate_sets):
    """
    Merge multiple CandidateSet objects by path, accumulating evidence.
    """
    fused_paths = set()
    fused_evidence = {}
    fused_source_counts = {}
    diagnostics = {"builders": {}}

    for cs in candidate_sets:
        if cs is None:
            continue

        diagnostics["builders"][cs.family] = cs.diagnostics or {}

        for k, v in (cs.source_counts or {}).items():
            fused_source_counts[k] = fused_source_counts.get(k, 0) + v

        for path in cs.candidate_paths:
            fused_paths.add(path)
            if path not in fused_evidence:
                fused_evidence[path] = {}
            fused_evidence[path].update(cs.evidence_by_path.get(path, {}))

    return list(fused_paths), fused_evidence, fused_source_counts, diagnostics
```

```python
def _build_candidate_sets_for_plan(self, plan, intent, project_meta, top_k):
    candidate_sets = []
    family = intent.family_hint or get_preset_family(plan.preset_id)

    if family == "type":
        if plan.preset_id == "documents":
            builder = DocumentCandidateBuilder(self.project_id)
            candidate_sets.append(builder.build(intent, project_meta, limit=top_k * 3))
        elif plan.preset_id == "screenshots":
            from services.candidate_builders.screenshot_candidate_builder import ScreenshotCandidateBuilder
            builder = ScreenshotCandidateBuilder(self.project_id)
            candidate_sets.append(builder.build(intent, project_meta, limit=top_k * 3))

    elif family == "people_event":
        builder = PeopleCandidateBuilder(self.project_id)
        candidate_sets.append(builder.build(intent, project_meta, limit=top_k * 3))

    return candidate_sets
```

```python
@dataclass
class ScoringWeights:
    """
    Deterministic scoring contract.

    S = w_clip * clip + w_recency * recency + w_fav * favorite
      + w_location * location + w_face * face_match
      + w_structural * structural + w_ocr * ocr

    w_recency: float = 0.05
    w_favorite: float = 0.08
    w_location: float = 0.04

    w_structural: float = 0.00
    w_ocr: float = 0.00
    w_event: float = 0.00

                self.w_clip /= total
                self.w_recency /= total
                self.w_favorite /= total
                self.w_location /= total
                self.w_face_match /= total
                self.w_structural /= total
                self.w_ocr /= total
                self.w_event /= total
```

```python
    def _build_candidate_set(
        self,
        intent: QueryIntent,
        project_meta: Dict[str, Dict],
```

```python
        """
        Dispatch to family-specific candidate builder if available.

        For type-family presets, route by preset, not only by family:
        - documents -> DocumentCandidateBuilder
        - screenshots -> ScreenshotCandidateBuilder
        """
        family = intent.family_hint
        if not family:
            return None

        try:
            if family == "type":
                if intent.preset_id == "screenshots":
                    builder = ScreenshotCandidateBuilder(self.project_id)
                else:
                    from services.candidate_builders.document_candidate_builder import DocumentCandidateBuilder
                    builder = DocumentCandidateBuilder(self.project_id)

                if builder_cls is None:
```

## 4.3 Add candidate fusion helper

Directly below `_build_candidate_set()`, add this new method:

```python
    def _fuse_candidate_sets(
        self,
        candidate_sets: List[CandidateSet],
    ):
        """
        Merge multiple CandidateSet objects by path and accumulate evidence.
        """
        fused_paths = set()
        fused_evidence = {}
        fused_source_counts = {}
        diagnostics = {"builders": {}}

        for cs in candidate_sets:
            if cs is None:
                continue

            diagnostics["builders"][cs.family] = cs.diagnostics or {}

            for k, v in (cs.source_counts or {}).items():
                fused_source_counts[k] = fused_source_counts.get(k, 0) + v

            for path in cs.candidate_paths:
                fused_paths.add(path)
                if path not in fused_evidence:
                    fused_evidence[path] = {}
                fused_evidence[path].update(cs.evidence_by_path.get(path, {}))

        return list(fused_paths), fused_evidence, fused_source_counts, diagnostics
```
